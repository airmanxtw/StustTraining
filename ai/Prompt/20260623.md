# 在這個目錄下,建立兩個專案
1. 在這個目錄下建立一個名為Demo20260623.sln的方案檔 
2. 建立一個C# WEBAPI的專案,目錄名稱為Demo20260623Api 
3. 建立一個C# MSTest的專案,目錄名稱為Demo20260623Test
4. 設定Demo20260623Test參考到Demo20260623Api專案,這是一個用來測試Demo20260623Api的專案
5. 將方案Demo20260623.sln分別引入Demo20260623Api及Demo20260623Test這兩個專案

# Demo20260623Api是用來提來提供WebApi給前端使用的專案,完成下列功能
1. 建立一個models目錄,這個目錄用來放資料結構的模型
2. 在models目錄下建立一個LoginModel.cs,宣告一個LoginModel的record,屬性有userid及password,皆為字串.
2. 建立一個/echo的router,用get的方式,單純簡單的回應"i'am ok"
3. 建立一個funs目錄,這個目錄用來放一些邏輯函式
4. 在funs目錄下,建立一個ldapService.cs,在裡面建立一個函式,傳入值是LoginModel,輸出值是boolean
5. 續第4點,不用真的連到ldap server,先假定userid是tester,密碼為12345,就回傳true,代表驗證成功
6. 建立Controllers時,不要用Minimal APIs的做法,請使用Class繼承ControllerBase的方式來建立router
    1. 在Controllers會用到的函式都使用依賴注入的方式注入到class裡使用,例如ldapService.cs
7. 建立一個/login的router,用post的方式傳入LoginModel,驗證成功後,回傳jwt token,有效期加30分鐘;並輸出另一份jwt token到cookie上,有效期加6小時
8. 續第7點,jwt toekn的取得及計算方式,由你自行決定,這只是一個展示系統
9. 建立一個/echoUserName的router,這個router必須經過token驗證才能使用,回傳登入者的userid
10. 建立一個/refreshtoken的router,這個router必須經過token驗證才能使用,並帶上第7點裡產生的cookie裡的token
11. 續10,驗證傳入的cookie token值是否有效,有效的話,重新再產生一份token並回傳

# Demo20260623Test是用來測試Demo20260623Api裡的router是否正確,完成下列功能
1. 用Microsoft.AspNetCore.TestHost測試Demo20260623Api裡所有的router
2. 續1,所有正向及負向情況都各測一次
3. 測試/login時,會涉及到ldapService.cs這個函式,在測試端建立一份,並用依賴注入方式注入到api裡.

# 在Demo20260623Api建立一個Public的目錄,並建立一個index.html檔案,使用html及javascript完成下列功能
1. 在請求router內容時,可參考Demo20260623Api的router內容,網址暫定https://localhost
2. 網頁會有兩個輸入欄,分別為帳號及密碼
3. 增加一個登入鈕,post到/login,並將取得token,展示在畫面上
4. 增加一個按鈕,按鈕顯示名稱為echoUserName,路由為/echoUserName,將第3點取得的token放入bearer中認證,將取得的資料展示在畫面上.
5. 增加一個按鈕,按鈕顯示名稱為refreshtoken,路由為/echoUserName,將token放入bearer認證,並傳入在第3點取得的cookie裡的token值
